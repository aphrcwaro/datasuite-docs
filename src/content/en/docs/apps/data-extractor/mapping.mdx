# Mapping Configuration

## Overview

Mapping defines **what data to extract** from DHIS2 and **how it is structured** for analysis.
It is the core step in the Data Extractor workflow in that it: fetches indicators of interest, 
maps/matches with corresponding DHIS2 data sources and standardizes outputs for Datasuite apps.

## 2. Creating a New Mapping



<Steps>
### Click **New Mapping**
![New Mapping](/images/extractor_mapping_newmap.jpg)

### Configure the mapping 
Define the mapping name, Description and the mapping mode
<Callout type="important">
  **Important:** Use a clear naming convention
  Example:
    Mapping name: CAM2026_CountryName
    Description: MNH Indicators for CAM 2026 Workshop
</Callout>

### Select the Mapping mode
There are currently two mapping modes:
- Countdown → This mode has a predefined list of MNCH indicators and structure as in the [harmonized data template] (/en/resources)
- Custom → Fully user-defined configuration, where the user can download any data that they have access rights to

<Callout type="warning">
  **Warning:** Changing the mapping mode will reset your progress if the mapping is not saved.
     Always save your mapping before switching modes
</Callout>
